## Context

Tab 2 renders the consolidated `sources.md` with markdown formatting
and scrollable heading anchors. This is the tab a `<SourceRef>` chip
deep-links into (task 0097).

See spec §5.

## Objectives

1. Add `react-markdown` and `remark-gfm` to `site/package.json` (or
   their currently idiomatic equivalents — document the choice in
   the task's PR description).
2. Create `site/components/transparency/DocumentTab.tsx`:
   - Fetches `/candidates/<id>/<version>/sources.md` on tab
     activation.
   - Computes SHA256 of the fetched bytes client-side (Web Crypto
     API, `crypto.subtle.digest`).
   - Renders a small header panel:
     - candidate name + version date (from props)
     - `human_review_completed` badge (from `metadata.json`)
     - computed SHA256 (full, monospace)
     - download link to the raw `.md`
   - Renders the markdown body via `react-markdown` with GFM
     enabled.
   - Every `<h2>` / `<h3>` renders with a stable slug `id`
     attribute generated by a pure `slugify(text)` helper placed
     in `site/lib/slug.ts` (minimal, no external dep).
3. Anchor scroll:
   - On mount (and on hash change), if the hash carries
     `anchor=<slug>`, scroll the matching heading into view with
     `behavior: "smooth"`, `block: "start"`.
4. Unit tests:
   - Markdown headings get slug ids (`## Retraites` → `id="retraites"`).
   - `slugify` handles accents, punctuation, spaces correctly.
   - Header panel renders SHA256 + review badge.
   - Tab behavior when sources.md fetch fails (error state copy).

## Acceptance Criteria

- [ ] `DocumentTab` renders `test-omega`'s `sources.md` with
      markdown formatting.
- [ ] Every `h2` / `h3` has a slug `id` matching `slugify(text)`.
- [ ] Deep-link `#transparence=document&anchor=<slug>` scrolls to
      the matching heading.
- [ ] SHA256 of the fetched bytes is displayed and computed
      client-side.
- [ ] Graceful error state when fetch fails.
- [ ] Tests pass: `pnpm --filter site test -- DocumentTab`.
- [ ] No lint / type errors.

## Hints for Agent

- Keep the `slugify` helper small and deterministic — normalize
  NFC → NFD → strip combining marks, lowercase, replace non
  `[a-z0-9]+` with `-`, trim dashes. Dozens of tested helpers
  exist online; write one, do not install `github-slugger` unless
  the bundle impact is trivial.
- Client-side SHA256 on files of this size (~20KB) is negligible.
- Do not enable `rehype-raw` — we do not render raw HTML from
  markdown.

## Editorial check

- [ ] No editing or summarization of `sources.md` — verbatim
      rendering only.
- [ ] No cardinal aggregation introduced.
