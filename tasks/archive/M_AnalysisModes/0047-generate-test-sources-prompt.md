## Context

To exercise the full pipeline without a real candidate, the operator
needs a copy-pasteable prompt they can drop into any web-chat LLM
(Gemini, ChatGPT, Claude) to produce a plausible but clearly fictional
French presidential program spanning all six dimension clusters.

## Objectives

1. Create `prompts/fixtures/generate-test-sources.md` with:
   - YAML frontmatter matching `prompts/README.md` format
     (`name: generate-test-sources`, `version: 0.1`, `status: stable`).
   - A short header explaining **this is not a pipeline prompt** —
     it's a human-facing template to paste into a chat UI.
   - The body of the prompt itself, in French, instructing the chat
     model to:
     - Generate a **clearly fictional** candidate program for the
       2027 French presidential election.
     - Cover all six analytical clusters from
       `docs/specs/analysis/dimensions.md` (Économie & finances,
       Social & démographie, Sécurité & souveraineté, Institutions &
       démocratie, Environnement & long terme, plus cross-cutting
       Intergénérationnel references).
     - At least 3 concrete measures per cluster.
     - At least 1 quantified claim per cluster (numbers, percentages,
       €/year) so the intergenerational audit and
       measurement-over-indictment facets have material to exercise.
     - Pick a **single coherent positioning** (e.g. "centre-droit
       souverainiste", "écosocialiste"), stated upfront, with
       self-consistent proposals.
     - Tag every section heading with `(fictif)` so the fictional
       nature is visually obvious.
     - Use the same top-level section structure as
       `prompts/consolidate-sources.md` output, so the generated
       content can be pasted directly as `sources.md`.
     - Include a mandatory header banner:
       ```
       > ⚠️ PROGRAMME FICTIF — généré pour tests de pipeline.
       > Ne reflète pas les positions d'une personne réelle.
       ```
     - **Avoid using real candidate names** as policy owners; may
       reference public figures only as positioning anchors (the same
       anchors used in `prompts/analyze-candidate.md` §9).
2. Add a short README snippet (within the same file or a sibling
   `prompts/fixtures/README.md`) explaining:
   - This folder holds **test fixtures**, not pipeline prompts.
   - Content generated by these prompts must land under a `test-*`
     candidate ID with `is_fictional: true`.
   - Link to spec section on test-candidate scaffolding.

## Acceptance Criteria

- [ ] `prompts/fixtures/generate-test-sources.md` exists with correct
      frontmatter
- [ ] Prompt body is in French, self-contained, and covers all
      dimension clusters
- [ ] Output format instructions align with the `sources.md`
      structure expected by the pipeline
- [ ] Fictional banner requirement is explicit and prominent
- [ ] `prompts/fixtures/` has its own README (or in-file section)
      explaining the folder's purpose
- [ ] `npm run lint` passes

## Hints for Agent

- Base the section skeleton on the template in
  `prompts/consolidate-sources.md`.
- Cite `docs/specs/analysis/dimensions.md` in the prompt so the chat
  model has a defined taxonomy to hit.
- Do NOT have the prompt try to produce analysis or positioning
  scores. It only produces the *source* document; analysis is a
  downstream step.

## Editorial check

- [ ] The prompt produces raw program content only, not analysis (no
      verdicts, no grades)
- [ ] The fictional banner is mandatory, not optional
- [ ] No instruction to mimic a real candidate — only to pick a
      coherent positioning

## Notes

This prompt is never sent programmatically. It is copy-pasted by the
operator. Keep it self-contained.
