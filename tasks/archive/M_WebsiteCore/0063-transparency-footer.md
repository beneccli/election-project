## Context

The full transparency drawer is deferred to M_Transparency. M_WebsiteCore
ships a **minimal footer** that already exposes the metadata structure a
sceptical reader needs: models used, exact versions, prompt SHA256s, human
review attestation, download links.

## Objectives

1. `site/components/transparency/TransparencyFooter.tsx` (server):
   - "Cette analyse a été produite par N modèles d'IA" line
   - Per-model pill: `provider` · `exact_version` · `status` ·
     `execution_mode` — from `metadata.analysis.models.<id>`
   - Two SHA256 hashes: `metadata.analysis.prompt_sha256` and
     `metadata.aggregation.prompt_sha256` — full 64-char hex rendered in
     monospace
   - Aggregator attestation: provider, exact_version, attested_by,
     attested_model_version, execution_mode
   - Human review status: green "Revue humaine terminée le <date> par
     <reviewer>" if `human_review_completed` is true, yellow "Revue humaine
     en cours" otherwise. Publish gate normally prevents the latter from
     reaching the site, but render defensively.
   - Download links (relative URLs pointing at static copies of the
     candidate artifacts):
     - `aggregated.json`
     - `metadata.json`
     - `raw-outputs/` listing (static index page — a simple HTML page with
       one link per file — left as a TODO note if not trivially achievable)
   - A `<details>` with label "Comment ces données ont été produites" that
     expands to a short paragraph linking to
     `/methodologie` (404 for now, will be filled by M_Methodology)
2. Files are exposed by **copying the candidate version directory into
   `site/public/candidates/<id>/<version>/`** as part of the build.
   Implement this as a small script `site/scripts/copy-candidate-artifacts.ts`
   invoked from the `build` script (`"build": "tsx scripts/copy-candidate-artifacts.ts && next build"`).
   The script:
   - Iterates `listCandidates()` (reusing the loader)
   - Copies each candidate's `current/aggregated.json`, `metadata.json`,
     `sources.md`, `raw-outputs/*.json`, `aggregation-notes.md` into
     `site/public/candidates/<id>/<version_date>/`
   - Skips `_manual/` and other scratch directories
   - Idempotent (deletes destination first)
3. Wire footer into `app/candidat/[id]/page.tsx`, replacing the 0057
   placeholder.

## Acceptance Criteria

- [ ] Footer renders for test-omega with all metadata fields populated
- [ ] Download links resolve to files inside `site/out/candidates/<id>/…`
      after build
- [ ] Prompt SHA256s render in full (not truncated) — copyable by the reader
- [ ] Human review status badge reflects metadata accurately
- [ ] `pnpm --filter site build` copies artifacts and succeeds

- `fs.cpSync(src, dest, { recursive: true, force: true, filter: (s) =>
  !s.includes("/_manual") })` handles the copy cleanly.
- The raw-outputs directory listing in v1 can be a simple static

- [ ] **Every prompt SHA256 is visible in full** — this is the verifiability
      hook. Do not truncate for aesthetics.

      alone; full `claude-opus-4-6` etc.)
- [ ] Human review status is never omitted — if review is incomplete, say so
