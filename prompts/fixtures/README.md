# prompts/fixtures/

> Test fixtures. **Not pipeline prompts.**

This folder holds human-facing copy-paste templates used to generate
synthetic content for pipeline testing. Nothing in here is loaded
programmatically by the election-project code.

## What belongs here

- Templates the operator pastes into a web-chat LLM (ChatGPT, Claude.ai,
  Gemini, Mistral Chat, Grok…) to produce fixture input for the
  pipeline.
- Clearly marked, human-read prompts. Frontmatter `status: stable`
  is acceptable, but `purpose:` must explicitly say "human-facing
  template, not a pipeline prompt".

## What does NOT belong here

- Any prompt loaded by scripts (`consolidate-sources.md`,
  `analyze-candidate.md`, `aggregate-analyses.md`) — those live in
  [`prompts/`](..).
- Any prompt whose wording changing would invalidate past analyses.
  Fixtures are regeneratable; canonical prompts are not.

## Contract

Content generated by any prompt in this folder:

1. Must be used only under a `test-*` candidate ID.
2. Must have `is_fictional: true` in `candidates/<test-id>/metadata.json`
   (set by `scripts/scaffold-candidate.ts --is-fictional`).
3. Must carry the `⚠️ PROGRAMME FICTIF` banner at the top of
   `sources.md`, and `(fictif)` suffixes on every section heading.
4. Can never become `current` without the `--allow-fictional` publish
   override.

See [`docs/specs/data-pipeline/analysis-modes.md`](../../docs/specs/data-pipeline/analysis-modes.md)
§ "Fictional candidates" for the full contract.
